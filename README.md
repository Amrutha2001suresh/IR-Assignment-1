# IR-Assignment-1
# End-to-End Information Retrieval System

## Overview

This project implements a complete Information Retrieval (IR) System using Python and Streamlit.

The system allows users to upload text documents, preprocess them, build multiple index structures, execute search queries, compare retrieval techniques, evaluate dictionary search structures, and perform tolerant retrieval using spelling correction based on Edit Distance.

The application provides an interactive user interface through Streamlit and demonstrates major concepts of Information Retrieval.

---

## Features Implemented

### 1. Document Upload

* Upload multiple `.txt` documents
* Display uploaded document contents
* Support multiple document collections

---

### 2. Text Preprocessing

The following preprocessing techniques are implemented:

* Lowercase conversion
* Tokenization
* Stopword removal
* Hyphen handling
* Stemming (Porter Stemmer)
* Lemmatization (WordNet Lemmatizer)

Users can choose between:

* No normalization
* Stemming
* Lemmatization

---

### 3. Inverted Index Construction

An inverted index is built from all uploaded documents.

The index stores:

Term → List of Document IDs

Example:

machine → [Doc1, Doc2, Doc3]

learning → [Doc1, Doc2, Doc5]

---

### 4. Biword Index Construction

A biword index is generated by storing adjacent word pairs.

Example:

machine learning

deep learning

artificial intelligence

The biword index supports phrase retrieval.

---

### 5. Positional Index Construction

A positional index stores:

Term → Document ID → Word Positions

Example:

machine

Doc1 → [4]

Doc2 → [8]

This enables exact phrase matching using term positions.

---

### 6. Query Processing

The system processes user queries and performs retrieval using:

* Inverted Index
* Biword Index
* Positional Index

Results are displayed side-by-side for comparison.

---

### 7. Phrase Query Comparison

Phrase queries are evaluated using:

* Biword Index
* Positional Index

A comparison table is displayed showing:

* Retrieved documents
* Accuracy comparison
* Retrieval behavior

---

### 8. BST vs B-Tree Dictionary Search

Dictionary lookup performance is compared using:

* Binary Search Tree (BST)
* B-Tree style Binary Search

For each query term:

* Search success is recorded
* Search execution time is measured

Inference is displayed to compare performance.

---

### 9. Tolerant Retrieval

Tolerant Retrieval is implemented using:

* Spelling Correction
* Edit Distance (Levenshtein Distance)

Examples:

machin → machine

learnng → learning

deeep → deep

The system automatically:

1. Detects misspelled terms
2. Finds closest vocabulary terms
3. Corrects the query
4. Performs retrieval again

The Tolerant Retrieval section is shown only when query correction is required.

---

### 10. Stemming vs Lemmatization Comparison

The system compares:

* Stemming
* Lemmatization

For each query:

* Processed query terms
* Retrieved document count
* Retrieved document list

are displayed and compared.
